# 要件定義書 — 渋谷 外貨両替レート比較サイト

## 1. 概要

### 1.1 目的
渋谷エリアの外貨両替店のレートを一覧で比較できるWebサイトを提供する。訪日外国人および日本人が、最も有利なレートの店舗を素早く見つけられるようにする。

### 1.2 本番URL
https://fx-shibuya.vercel.app

### 1.3 ターゲットユーザー
- 渋谷で外貨両替をしたい訪日外国人（英語・中国語・韓国語話者）
- 渋谷で外貨を購入したい日本人旅行者

---

## 2. 機能要件

### 2.1 レート比較表示
- 対象8通貨のレートを通貨ごとのカードで一覧表示する
- 各カードに市場レート・各店舗のsell/buyレート・乖離率を表示する
- 乖離率は色分けバッジで視覚的に判断可能にする
  - 緑（< 2%）: お得
  - 黄（2〜4%）: 普通
  - 赤（> 4%）: 割高

### 2.2 対象通貨
| コード | 通貨名 |
|--------|--------|
| USD | 米ドル |
| EUR | ユーロ |
| GBP | 英ポンド |
| AUD | 豪ドル |
| CNY | 中国元 |
| KRW | 韓国ウォン |
| HKD | 香港ドル |
| TWD | 台湾ドル |

### 2.3 対象店舗

| 店舗 | データ取得 | 取得可能データ | 備考 |
|------|-----------|-------------|------|
| Travelex（渋谷店 / Mark City店） | REST API | sell のみ | buy レートはAPI非公開 |
| World Currency Shop（渋谷道玄坂店） | HTMLスクレイピング | sell / buy | KRWは100単位表示（`annotationFlg: 1`）→ 1単位に正規化 |
| 大黒屋（質渋谷店） | 取得不可 | なし | 「レートは店頭にてご確認ください」と表示 |

### 2.4 レート用語定義
- **sell（販売レート）**: 顧客が外貨を買う際のレート（日本円 → 外貨）。顧客が支払う円額。
- **buy（買取レート）**: 顧客が外貨を売る際のレート（外貨 → 日本円）。顧客が受け取る円額。
- **乖離率（スプレッド）**: 店舗レートと市場仲値レートの差をパーセントで表したもの。

### 2.5 多言語対応
- 対応言語: 日本語（デフォルト）、英語、中国語（簡体字）、韓国語
- URL構造: `/{locale}` （例: `/ja`, `/en`, `/zh`, `/ko`）
- ブラウザ言語による自動リダイレクト
- 翻訳対象: UI全文、通貨名、店舗情報、免責事項

### 2.6 店舗情報
各店舗について以下を表示する:
- 店舗名（多言語）
- 住所（多言語）
- 営業時間
- 公式サイトへのリンク
- Google Mapsへのリンク

### 2.7 レート更新
- ISR（Incremental Static Regeneration）により15分間隔で自動更新
- 市場レートは1時間キャッシュ
- 最終更新時刻をページ上部に表示（JST）

### 2.8 エラーハンドリング
- 一部データソースが失敗しても残りのデータで描画する（グレースフルデグラデーション）
- 全失敗時はエラーページを表示し、リトライボタンを提供する
- 取得不可の店舗/通貨は「取得不可」または「取扱なし」と表示する

## 4. データ取得仕様

### 4.1 市場レート（ExchangeRate-API）
- エンドポイント: `GET https://open.er-api.com/v6/latest/JPY`
- レスポンス: JPYベースの為替レート（1 JPY = x 外貨）
- 変換: 逆数をとって「1外貨 = ○円」に変換
- キャッシュ: `revalidate: 3600`（1時間）

### 4.2 Travelex（REST API）
- エンドポイント: `GET https://api.travelex.net/salt/rates/current?key=Travelex&site=%2FJAJP`
- レスポンス: `{ foreignCurrencyAsBase: true, rates: { USD: 161.22, ... }, lastModified: "..." }`
- ratesの値は「1外貨 = ○円」の販売レート（sell）
- 買取レート（buy）はAPI非公開のため `null`
- キャッシュ: `revalidate: 900`（15分）

### 4.3 World Currency Shop（HTMLスクレイピング）
- URL: `GET https://www.tokyo-card.co.jp/wcs/rate.php`
- 抽出対象: HTML内の `window.context.rate['通貨コード']` JavaScript変数
- パース形式:
  ```
  window.context.rate[ 'USD' ] = {
    currencyName: 'ドル',
    annotationFlg: 0,     // 1の場合は100単位表示
    sell: { cash: 159.60, tc: 157.60 },
    buy: { cash: 153.10, tc: 155.20 }
  };
  ```
- cashの値を使用（tc = トラベラーズチェック、非対象）
- `annotationFlg: 1` の場合（KRW）: 100で割って1通貨単位に正規化
- `999.00` は「取扱なし」として除外
- キャッシュ: `revalidate: 900`（15分）

### 4.4 オーケストレーション
- `Promise.allSettled` で3ソースを並列取得
- 一部失敗時: 成功したデータのみで描画
- 全失敗時: エラーページ表示

## 6. リスクと対策

| リスク | 影響 | 対策 | 現状 |
|--------|------|------|------|
| Travelex APIの仕様変更・停止 | sell レート取得不可 | エラー時は「取得不可」表示。API URLはコード内で管理 | 稼働中 |
| WCSのHTML構造変更 | sell/buy レート取得不可 | 正規表現を修正。構造はシンプルなので対応容易 | 稼働中 |
| ExchangeRate-API無料枠超過 | 市場レート取得不可 | 1時間キャッシュ。月1,500回で十分（15分更新 = 月約2,880回だがISRキャッシュで実際はそれ以下） | 稼働中 |
| Travelex buyレートが非公開 | buy比較不可 | sell のみ表示。UIで「取扱なし」と表記 | 既知の制約 |
